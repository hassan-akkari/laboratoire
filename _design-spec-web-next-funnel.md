# web-next funnel — design spec (synth of editorial+bento winners)

## northStar

Editorial booking funnel on the warm-indigo DARK identity: an editorial-grid premium experiences brand where oversized crisp Space Grotesk display type, large image/placeholder "plates", a strict 4px spacing rhythm, and viewport-filling stages make every screen read as a deliberate product step instead of a dev-jargon capability checklist. Base = Editorial (top judge totals). Grafts: a single hero featured-experience plate + uniform card grid rhythm (from bento), restrained monospace as a deliberate craft accent capped at one slot per screen (from Engine Room), price-forward card footer with an honestly-labelled secondary action (from Conversion). Hard rule: fix the heading-blur, dead-space, weak-input, and top-sliver bugs at the SYSTEM level in globals.css so the polish is uniform across all 8 pages; never introduce a new hue (derive every accent from --app-accent); never exceed 14px radius on rectangular surfaces; keep all form fields on raw .field CSS (AppInput/AppSelect/AppTextarea are still v2 and would render unstyled).

## depthTokens

On-identity, dark, low-key (never glossy) elevation — pair EVERY shadow with a visible 1px border because shadow alone is invisible on #080808. :root tokens: --elev-0:none; --elev-1:0 1px 0 color-mix(in srgb,#fff 4%,transparent) inset, 0 2px 8px -4px rgba(0,0,0,.55) (resting fields/flat tiles rely on border, this is the minimal lift); --elev-2:0 1px 0 color-mix(in srgb,#fff 5%,transparent) inset, 0 12px 28px -18px rgba(0,0,0,.62) (resting cards); --elev-3:0 1px 0 color-mix(in srgb,#fff 6%,transparent) inset, 0 22px 44px -26px rgba(0,0,0,.7) (hero/feature panels, hovered cards). BORDERS: raise the washed-out default from color-mix(--app-border 74%,transparent) to a visible --app-border-strong:color-mix(in srgb,var(--app-fg) 12%,var(--app-border)) for cards/tiles so edges read against #151515 and #080808; interactive surfaces (fields, hovered cards) step to color-mix(in srgb,var(--app-fg) 22%,var(--app-border)); hover/active accent step to color-mix(in srgb,var(--app-accent) 40%,var(--app-border)). FOCUS RING: introduce --app-accent-focus:#6b6f93 (brightened from the low-luma #4a4e69) used as the focus border, plus the 3px --app-accent-soft halo — the halo alone is decorative (24% alpha), so the visible-contrast comes from the --app-accent-focus border (this fixes the judges' WCAG 2.4.7 concern). RADII (tight, honor the lib's baked values): --r-sm:8px (AppButton default, small buttons, chips-as-pill stay 999px); --r-field:10px (inputs); --r-md:12px (secondary cards, notices); --r-lg:14px (AppCard lg, experience cards, hero, media plates, sticky summary). NEVER exceed 14px on rectangular surfaces — explicitly downsize the current .hero-card 24px (globals.css:137), .card 20px, .app-nav 20px, .stat-card 18px, tables 18px all into the 10-14px band. No drop shadows on chips/fields (border + fill only). AppButton/AppCard map radius via inline style (sm=8/md=12/lg=14); AppChip radius is a no-op so chips keep their pill shape — accepted.
